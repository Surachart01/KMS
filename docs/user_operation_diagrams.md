## 2. แผนภาพลำดับการทำงาน (Operation Flows)

### 2.1 ขั้นตอนการเบิกกุญแจ (Borrowing Flow)
```mermaid
flowchart TD
    A["เริ่มต้น"] --> B{"เลือกห้อง/กุญแจ"}
    B --> C["สแกนใบหน้า / ยืนยันตัวตน"]
    C --> D{"มีสิทธิ์ตามตาราง?"}
    D -- "ไม่มี" -- > E["ระบุเหตุผลการเบิกกรณีพิเศษ"]
    D -- "มี" --> F["ระบบ: สร้างรายการเบิก (Booking)"]
    E --> F
    F --> G["ฮาร์ดแวร์: ปลดล็อคกลอนไฟฟ้า"]
    G --> H["ผู้ใช้ออกกุญแจออกจากตู้"]
    H --> I["สิ้นสุด: ไฟสีเขียวติดค้าง"]
```

### 2.2 ขั้นตอนการคืนกุญแจ (Returning Flow)
```mermaid
flowchart TD
    A["เริ่มต้น"] --> B["สแกนใบหน้า / ยืนยันตัวตน"]
    B --> C{"พบรายการเบิกที่ค้างอยู่?"}
    C -- "ไม่พบ" --> D["แสดงข้อผิดพลาด"]
    C -- "พบ" --> E["ยืนยันข้อมูลการคืน"]
    E --> F["ระบบ: รอการเสียบกุญแจ"]
    F --> G["ผู้ใช้เสียบกุญแจลงในช่อง"]
    G --> H{"พบ NFC และข้อมูลตรงกัน?"}
    H -- "ตรงกัน" --> I["ระบบ: ปิดรายการเบิก"]
    I --> J["สิ้นสุด: ไฟสีเขียวติดค้าง"]
    H -- "ไม่ตรง/หมดเวลา" --> K["การคืนไม่สำเร็จ / แจ้งเตือน"]
```

### 2.3 ขั้นตอนการสลับสิทธิ์ (Swap Rights Flow)
*ใช้ในกรณีผู้ใช้ 2 คนต้องการแลกเปลี่ยนกุญแจที่ถืออยู่โดยไม่ต้องนำกุญแจมาคืนที่ตู้*
```mermaid
flowchart TD
    A["ผู้ใช้ ก + ผู้ใช้ ข พบกันที่ตู้"] --> B["เลือกเมนู 'สลับสิทธิ์'"]
    B --> C["ผู้ใช้ ก: สแกนใบหน้า"]
    C --> D["ผู้ใช้ ข: สแกนใบหน้า"]
    D --> E{"ตรวจสอบความเป็นเจ้าของกุญแจทั้งคู่?"}
    E -- "ถูกต้อง" --> F["ระบบ: สลับชื่อผู้รับผิดชอบกุญแจ"]
    F --> G["SystemLogs: บันทึกประวัติการสลับ"]
    G --> H["สิ้นสุด: สลับสิทธิ์สำเร็จ"]
```

### 2.4 ขั้นตอนการโอนสิทธิ์ (Transfer Rights Flow)
*ใช้ในกรณีผู้มีชื่อเบิกต้องการโอนความรับผิดชอบกุญแจให้ผู้อื่น*
```mermaid
flowchart TD
    A["ผู้โอน: สแกนใบหน้า"] --> B["เลือกเมนู 'โอนสิทธิ์'"]
    B --> C["ผู้รับโอน: สแกนใบหน้า"]
    C --> D{"ตรวจสอบสถานะกุญแจของผู้โอน?"}
    D -- "ถูกต้อง" --> E["ระบบ: เปลี่ยนชื่อผู้รับผิดชอบเป็นผู้รับโอน"]
    E --> F["SystemLogs: บันทึกประวัติการโอน"]
    F --> G["สิ้นสุด: โอนสิทธิ์สำเร็จ"]
```
