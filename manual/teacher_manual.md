# ภาคผนวก ก-2: คู่มือการใช้งานระบบจัดการกุญแจสำหรับอาจารย์

## 1. บทนำและบทบาทของอาจารย์

อาจารย์มีบทบาทสำคัญในระบบจัดการกุญแจในฐานะผู้สร้างและบริหารตารางสอน ซึ่งตารางสอนนั้นเองเป็นกลไกหลักที่กำหนดสิทธิ์การเบิกกุญแจห้องของนักศึกษา อาจารย์ยังมีสิทธิ์พิเศษที่สูงกว่านักศึกษาในการเบิกกุญแจหน้าตู้ด้วย

อาจารย์สามารถใช้งานระบบจัดการกุญแจได้ 2 ช่องทาง ได้แก่

ช่องทางที่ 1 เครื่อง Kiosk หน้าตู้กุญแจ สำหรับเบิกกุญแจ คืนกุญแจ โอนสิทธิ์ สลับสิทธิ์ และย้ายกุญแจ

ช่องทางที่ 2 Web Dashboard สำหรับจัดการตารางสอน นำเข้าข้อมูลจาก Excel และแก้ไขโปรไฟล์

อาจารย์มีสิทธิ์เบิกกุญแจได้แม้ไม่มีตารางสอนในช่วงเวลานั้น ซึ่งต่างจากนักศึกษาที่ต้องมีตารางเรียนตรงกับวันและเวลา รายการเบิกทุกรายการของอาจารย์จะถูกบันทึกลงในประวัติการทำรายการโดยอัตโนมัติ

## 2. การเข้าสู่ระบบ Web Dashboard

### 2.1 ขั้นตอนเข้าสู่ระบบ

เปิดเว็บเบราว์เซอร์ Google Chrome หรือ Microsoft Edge เวอร์ชันล่าสุด แนะนำให้ใช้เบราว์เซอร์ที่รองรับ JavaScript เวอร์ชันใหม่ เนื่องจาก Web Dashboard พัฒนาด้วย Next.js พิมพ์ URL ของระบบจัดการกุญแจในแถบที่อยู่ หน้าจอแสดงหน้าเข้าสู่ระบบ KMS พร้อมแบบฟอร์มล็อกอิน

กรอกอีเมลของอาจารย์ในช่องอีเมล และรหัสผ่านในช่องรหัสผ่าน อีเมลและรหัสผ่านเริ่มต้นจะได้รับจากเจ้าหน้าที่ระบบ กดปุ่มเข้าสู่ระบบ ระบบตรวจสอบข้อมูลและนำทางไปยังหน้า Dashboard ของอาจารย์

### 2.2 เมนูที่ใช้งานได้

หลังเข้าสู่ระบบสำเร็จ แถบเมนูทางด้านซ้ายจะแสดงเมนูที่อาจารย์สามารถใช้งานได้ 2 รายการ ได้แก่ เมนูตารางสอนของฉัน สำหรับดู เพิ่ม แก้ไข ลบ และนำเข้าตารางสอน และเมนูโปรไฟล์ สำหรับดูและแก้ไขข้อมูลส่วนตัวรวมถึงเปลี่ยนรหัสผ่าน

การออกจากระบบอย่างปลอดภัยทำได้โดยคลิกที่ชื่อผู้ใช้มุมขวาบน แล้วเลือกรายการออกจากระบบ

## 3. หน้าตารางสอนของฉัน

หน้าตารางสอนของฉันเป็นหน้าหลักที่อาจารย์ใช้จัดการข้อมูลตารางสอน ซึ่งตารางสอนเป็นกลไกสำคัญที่กำหนดว่านักศึกษาคนใดมีสิทธิ์เบิกกุญแจห้องใดได้บ้างในแต่ละวัน

### 3.1 ภาพรวมหน้าตารางสอน

หน้าจอประกอบด้วย 3 ส่วนหลัก ดังนี้

ส่วนที่ 1 ส่วนหัวแสดงข้อมูลอาจารย์ ได้แก่ ชื่ออาจารย์ นามสกุล และจำนวนวิชาที่สอนทั้งหมดในระบบ

ส่วนที่ 2 การ์ดสรุปรายวิชา แสดงการ์ดสำหรับรายวิชาที่อาจารย์รับผิดชอบ แต่ละการ์ดแสดงรหัสวิชา ชื่อวิชา และจำนวนคาบสอนในตาราง

ส่วนที่ 3 รายการตารางสอนทั้งหมดที่อาจารย์สร้างไว้ เรียงตามวันในสัปดาห์ แต่ละรายการแสดงวัน เวลาเริ่มต้นและเวลาสิ้นสุด รหัสและชื่อวิชา รหัสห้อง จำนวนนักศึกษา พร้อมปุ่มแก้ไขและลบ

### 3.2 การเพิ่มตารางสอนใหม่

**เงื่อนไขก่อนใช้งาน**

อาจารย์ต้องมีรายวิชาในระบบอย่างน้อย 1 รายวิชาก่อนจึงจะสามารถเพิ่มตารางสอนได้ หากยังไม่มีรายวิชา ปุ่มเพิ่มตารางเรียนจะไม่สามารถกดได้และระบบจะแสดงข้อความแจ้งให้ติดต่อเจ้าหน้าที่เพื่อเพิ่มรายวิชาก่อน

**ขั้นตอนการเพิ่มตารางสอน**

กดปุ่มเพิ่มตารางเรียนซึ่งอยู่มุมขวาบนของหน้าจอ หน้าต่างแบบฟอร์มจะเปิดขึ้นทางด้านขวาของหน้าจอ แบบฟอร์มประกอบด้วย 2 ส่วนหลัก

ส่วนที่ 1 ข้อมูลตารางเรียน อาจารย์ต้องเลือกรายวิชาจากเมนูดรอปดาวน์ที่แสดงเฉพาะรายวิชาที่อาจารย์รับผิดชอบ สามารถพิมพ์รหัสวิชาหรือชื่อวิชาในช่องเพื่อค้นหาได้ ถัดมาต้องกรอกรหัสห้องเรียน เช่น 44-703 โดยระบบแปลงเป็นตัวพิมพ์ใหญ่อัตโนมัติ รหัสห้องต้องตรงกับรหัสที่เจ้าหน้าที่กำหนดไว้ในหน้าจัดการกุญแจ จากนั้นเลือกวันในสัปดาห์ตั้งแต่วันอาทิตย์ถึงวันเสาร์ และเลือกเวลาเริ่มกับเวลาสิ้นสุดผ่านตัวเลือกเวลาแบบนาฬิกาที่ปรับได้ทีละ 5 นาที

ส่วนที่ 2 รายชื่อนักศึกษา ระบบแสดงช่องค้นหาที่จัดกลุ่มนักศึกษาตามสาขาวิชาและกลุ่มเรียน อาจารย์สามารถค้นหานักศึกษาได้โดยพิมพ์รหัสหรือชื่อ เลือกจากรายการที่ปรากฏ รายชื่อที่เลือกจะปรากฏในตารางด้านล่างพร้อมรหัสนักศึกษา ชื่อ นามสกุล สาขา และกลุ่มเรียน สามารถลบนักศึกษาออกรายบุคคลได้โดยกดปุ่มลบสีแดงด้านขวา

หลังตรวจสอบข้อมูลทั้งหมดแล้ว กดปุ่มเพิ่มเพื่อบันทึก ระบบแจ้งผลสำเร็จและรายการใหม่ปรากฏในตารางทันที

### 3.3 การแก้ไขตารางสอน

กดปุ่มแก้ไขในแถวของตารางสอนที่ต้องการ แบบฟอร์มแก้ไขเปิดขึ้นพร้อมข้อมูลเดิมทั้งหมด แก้ไขข้อมูลที่ต้องการ เช่น เปลี่ยนห้อง เปลี่ยนเวลา หรืออัปเดตรายชื่อนักศึกษา แล้วกดปุ่มบันทึก

การเปลี่ยนห้องหรือเวลาในตารางสอนจะมีผลต่อสิทธิ์การเบิกกุญแจของนักศึกษาในวันถัดไปหลังจากที่เจ้าหน้าที่ซิงค์ตาราง สิทธิ์ที่ซิงค์ไปแล้วในวันปัจจุบันจะไม่เปลี่ยนแปลงย้อนหลัง

### 3.4 การลบตารางสอน

กดปุ่มลบในแถวที่ต้องการลบ กล่องยืนยันการดำเนินการปรากฏขึ้น กดปุ่มลบเพื่อยืนยัน หรือกดปุ่มยกเลิกเพื่อยกเลิกการลบ

การลบตารางสอนจะส่งผลให้นักศึกษาในกลุ่มเรียนนั้นสูญเสียสิทธิ์เบิกกุญแจห้องดังกล่าวในการซิงค์ครั้งถัดไป ควรพิจารณาให้รอบคอบก่อนดำเนินการ

### 3.5 การนำเข้าตารางเรียนจากไฟล์ Excel

ระบบรองรับการนำเข้าข้อมูลตารางเรียนจากไฟล์ Excel ในรูปแบบรายชื่อนักศึกษาประจำวิชาที่ออกโดยระบบทะเบียนนักศึกษาของมหาวิทยาลัย การนำเข้าช่วยลดเวลาในการกรอกข้อมูลและลดโอกาสเกิดข้อผิดพลาด

กดปุ่มนำเข้าจาก Excel ซึ่งแสดงด้วยไอคอนไฟล์ Excel สีเขียว หน้าต่างเลือกไฟล์ของระบบปฏิบัติการจะเปิดขึ้น เลือกไฟล์ .xlsx หรือ .xls ที่ต้องการนำเข้า

ระบบอ่านไฟล์และวิเคราะห์ข้อมูลโดยอัตโนมัติ กระบวนการนี้ใช้เวลาไม่เกิน 5 วินาที ระบบจะดึงข้อมูลดังนี้ รหัสวิชาและชื่อวิชาจากบรรทัดที่ 4 ของไฟล์ วัน เวลา และรหัสห้องเรียนจากบรรทัดที่ 5 และรายชื่อนักศึกษาจากบรรทัดที่ 9 เป็นต้นไป

หน้าต่างยืนยันการนำเข้าเปิดขึ้นพร้อมแสดงข้อมูลที่อ่านได้ทั้งหมด อาจารย์สามารถแก้ไขรหัสวิชา ชื่อวิชา วัน เวลา ห้องเรียน รวมถึงเพิ่มหรือลบนักศึกษาออกจากรายการได้ก่อนยืนยัน กดปุ่มนำเข้าข้อมูลเพื่อยืนยัน ระบบแจ้งผลสำเร็จและรายการปรากฏในตาราง

## 4. การใช้งานเครื่อง Kiosk สำหรับอาจารย์

อาจารย์ใช้งานเครื่อง Kiosk โดยสแกนใบหน้าผ่านกล้อง ZKTeco เพื่อยืนยันตัวตน เมื่อระบบจำแนกได้ว่าผู้ใช้เป็นอาจารย์ หน้าจอจะแสดงเมนูครบทุกฟังก์ชัน อาจารย์สามารถเบิกกุญแจห้องใดก็ได้ที่กุญแจว่างอยู่ แม้ไม่มีตารางสอนในช่วงเวลานั้น รายการดังกล่าวจะถูกบันทึกลงในประวัติการทำรายการโดยอัตโนมัติ

รายละเอียดขั้นตอนการใช้งานฟังก์ชันทั้งหมด ให้อ้างอิงตามขั้นตอนเดียวกับในภาคผนวก ก-1 เนื่องจากขั้นตอนและหน้าจอเหมือนกันทุกประการ เพียงแต่ระบบจะมอบสิทธิ์พิเศษเพิ่มเติมในกรณีที่วิเคราะห์ได้ว่าผู้ใช้คืออาจารย์

## 5. การแก้ไขโปรไฟล์

คลิกที่ชื่อผู้ใช้มุมขวาบน หรือเลือกเมนูโปรไฟล์ในแถบเมนูด้านซ้าย ระบบแสดงข้อมูลส่วนตัวปัจจุบัน ได้แก่ ชื่อและนามสกุล อีเมล รหัสประจำตัวอาจารย์ และวันที่สร้างบัญชี

ในส่วนเปลี่ยนรหัสผ่าน กรอกรหัสผ่านปัจจุบันในช่องแรก กรอกรหัสผ่านใหม่ในช่องที่สอง และกรอกรหัสผ่านใหม่อีกครั้งในช่องยืนยัน รหัสผ่านควรมีความยาวอย่างน้อย 8 ตัวอักษร กดปุ่มบันทึก ระบบยืนยันและอัปเดตรหัสผ่านทันที

## 6. กระบวนการทำงานร่วมกับเจ้าหน้าที่

### 6.1 ลำดับขั้นตอนที่นักศึกษาสามารถเบิกกุญแจได้

เพื่อให้นักศึกษาสามารถเบิกกุญแจตามตารางที่อาจารย์สร้างไว้ได้ กระบวนการต้องครบถ้วนตามลำดับดังนี้

ลำดับที่ 1 อาจารย์สร้างหรืออัปเดตตารางสอนใน Web Dashboard พร้อมระบุรายชื่อนักศึกษาให้ครบ

ลำดับที่ 2 เจ้าหน้าที่ซิงค์ตารางเรียนในหน้าอนุญาตเบิกกุญแจทุกเช้าของวันทำการ กระบวนการนี้อ่านตารางที่อาจารย์สร้างไว้และสร้างสิทธิ์รายวันอัตโนมัติ

ลำดับที่ 3 นักศึกษามาเบิกกุญแจที่เครื่อง Kiosk ได้ในช่วงเวลาที่ระบุในตาราง

### 6.2 กรณีที่ต้องประสานงานเร่งด่วน

กรณีที่อาจารย์เปลี่ยนแปลงตารางสอนในวันนั้นเอง เจ้าหน้าที่ต้องซิงค์ข้อมูลใหม่หลังจากที่อาจารย์บันทึกการเปลี่ยนแปลงแล้ว

กรณีที่มีนักศึกษาต้องใช้ห้องเร่งด่วนโดยไม่มีในตาราง เจ้าหน้าที่สามารถเพิ่มสิทธิ์แบบกำหนดเองได้ทันที โดยอาจารย์แจ้งชื่อนักศึกษาและช่วงเวลาที่ต้องการ

## 7. การตรวจสอบผลการนำเข้าจาก Excel

### 7.1 รูปแบบไฟล์ Excel ที่รองรับ

ระบบ KMS รองรับการนำเข้าข้อมูลจากไฟล์ Excel รูปแบบ .xlsx และ .xls ที่ออกโดยระบบทะเบียนนักศึกษาของมหาวิทยาลัยโดยเฉพาะ ไม่ควรแก้ไขโครงสร้างไฟล์ก่อนนำเข้า เนื่องจากระบบอ่านตามตำแหน่งที่กำหนด

หากมีการแก้ไขจำเป็น เช่น เปลี่ยนรหัสห้อง ให้ดำเนินการในหน้าต่างยืนยันการนำเข้าที่ระบบแสดงขึ้นแทน

### 7.2 การตรวจสอบผลการนำเข้า

หลังนำเข้าสำเร็จ ให้ตรวจสอบรายการดังนี้

ตรวจสอบที่ 1 จำนวนนักศึกษาในตารางตรงกับจำนวนที่ลงทะเบียนไว้ หากไม่ตรง อาจมีนักศึกษาบางคนที่รหัสไม่ตรงกับฐานข้อมูล ให้เพิ่มเองด้วยตนเอง

ตรวจสอบที่ 2 รหัสห้องถูกต้องและตรงกับห้องที่จะใช้จริง

ตรวจสอบที่ 3 วัน เวลาเริ่ม และเวลาสิ้นสุดถูกต้องครบถ้วน
