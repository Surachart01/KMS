# ภาคผนวก ก-3: คู่มือการใช้งานระบบจัดการกุญแจสำหรับเจ้าหน้าที่

## 1. บทนำและบทบาทของเจ้าหน้าที่

เจ้าหน้าที่คือผู้ดูแลระบบในระดับปฏิบัติการ มีสิทธิ์การเข้าถึงระบบในระดับสูงสุดครอบคลุมทุกส่วนของ Web Dashboard รวมถึงการจัดการฮาร์ดแวร์ตู้กุญแจผ่านระบบ เจ้าหน้าที่ทำหน้าที่เป็นผู้เชื่อมโยงระหว่างฮาร์ดแวร์ตู้กุญแจ ระบบซอฟต์แวร์ และผู้ใช้งานทุกกลุ่ม

เจ้าหน้าที่สามารถใช้งานระบบได้ 2 ช่องทาง ได้แก่

ช่องทางที่ 1 Web Dashboard สำหรับบริหารจัดการฐานข้อมูลและการดำเนินงานของระบบทั้งหมด

ช่องทางที่ 2 เครื่อง Kiosk หน้าตู้กุญแจ สำหรับเบิกกุญแจ คืนกุญแจ โอนสิทธิ์ สลับสิทธิ์ และย้ายกุญแจ

### 1.1 ภาพรวมเมนูใน Web Dashboard

Web Dashboard ของเจ้าหน้าที่ประกอบด้วยเมนูหลักดังต่อไปนี้

เมนู Dashboard ใช้ดูภาพรวมสถิติทั้งหมดของระบบ เมนูข้อมูลหลักซึ่งแบ่งเป็น 4 หน้าย่อย ได้แก่ สาขาวิชา กลุ่มเรียน รายวิชา และเหตุผลการเบิก เมนูตารางเรียนสำหรับนำเข้าและจัดการตารางเรียน เมนูจัดการกุญแจสำหรับกำหนดข้อมูลกุญแจแต่ละดอก เมนูอนุญาตเบิกกุญแจสำหรับจัดการสิทธิ์รายวัน เมนูจัดการผู้ใช้งานซึ่งแบ่งเป็น 3 หน้าย่อย ได้แก่ นักศึกษา อาจารย์ และเจ้าหน้าที่ เมนูประวัติการเบิก-คืนสำหรับดูและแก้ไขประวัติทั้งหมด และเมนูจัดการ Penalty สำหรับดูและจัดการบทลงโทษ

## 2. การเข้าสู่ระบบ Web Dashboard

### 2.1 ขั้นตอนเข้าสู่ระบบ

เปิดเว็บเบราว์เซอร์และพิมพ์ URL ของระบบจัดการกุญแจในช่องที่อยู่เว็บ กรอกอีเมลและรหัสผ่านของเจ้าหน้าที่ในช่องที่กำหนด กดปุ่มเข้าสู่ระบบ ระบบตรวจสอบสิทธิ์และนำทางสู่หน้า Dashboard ของเจ้าหน้าที่

### 2.2 ความปลอดภัย

เจ้าหน้าที่ควรปฏิบัติตามข้อกำหนดด้านความปลอดภัยดังนี้

ไม่เปิดเผยรหัสผ่านให้ผู้อื่นทราบ เนื่องจากบัญชีเจ้าหน้าที่มีสิทธิ์สูงและสามารถดำเนินการที่มีผลต่อข้อมูลสำคัญได้

ออกจากระบบทุกครั้งหลังใช้งานเสร็จ โดยเฉพาะเมื่อใช้คอมพิวเตอร์สาธารณะ การออกจากระบบทำได้โดยคลิกที่ชื่อผู้ใช้มุมขวาบนและเลือกรายการออกจากระบบ

เปลี่ยนรหัสผ่านเป็นประจำ และใช้รหัสผ่านที่มีความซับซ้อนเพียงพอ

## 3. หน้า Dashboard

หน้า Dashboard เป็นหน้าแรกที่เจ้าหน้าที่เห็นหลังเข้าสู่ระบบ ออกแบบมาเพื่อให้เจ้าหน้าที่ทราบสถานะปัจจุบันของระบบได้ทันที

![ภาพที่ ก.3-1 หน้า Dashboard ของเจ้าหน้าที่ แสดงสถิติภาพรวม](/Users/surachartlimrattanaphun/.gemini/antigravity/brain/ae421eca-c2bb-4a49-9e4a-94a01b6e3aed/kms_dashboard_1776652718829.png)

### 3.1 การ์ดสถิติหลัก

ส่วนบนสุดของหน้า Dashboard แสดงการ์ดข้อมูลสำคัญ 4 ใบ ดังนี้

การ์ดกุญแจทั้งหมด แสดงจำนวนกุญแจทุกดอกที่บันทึกไว้ในระบบ ช่วยให้ทราบขนาดของระบบโดยรวม

การ์ดพร้อมใช้งาน แสดงจำนวนกุญแจที่อยู่ในตู้และพร้อมให้เบิก ช่วยให้ทราบว่ามีกุญแจว่างเท่าใด

การ์ดกำลังถูกยืม แสดงจำนวนกุญแจที่ถูกยืมออกไปในขณะนั้น ช่วยตรวจสอบการใช้งานแบบเรียลไทม์

การ์ดตารางเรียน แสดงจำนวนตารางเรียนทั้งหมดในระบบ

### 3.2 ข้อมูลสถิติเพิ่มเติม

นอกจากการ์ดสถิติหลักแล้ว หน้า Dashboard ยังแสดงข้อมูลเพิ่มเติมอีก 3 ส่วน ดังนี้

ส่วนที่ 1 สถิติจำนวนผู้ใช้งาน แสดงจำนวนนักศึกษาและอาจารย์ที่มีบัญชีในระบบทั้งหมด เพื่อให้เจ้าหน้าที่ทราบขนาดผู้ใช้งานโดยรวม

ส่วนที่ 2 สถิติกุญแจแยกรายตึก แสดงจำนวนกุญแจของแต่ละอาคารโดยแบ่งเป็นกุญแจที่พร้อมให้เบิกและกุญแจที่ถูกยืมออกไปในขณะนั้น

ส่วนที่ 3 ตารางการยืม-คืนล่าสุด แสดงรายการทำรายการล่าสุด 10 รายการ ประกอบด้วยชื่อผู้ยืม รหัสห้อง สถานะ และเวลา ใช้สำหรับตรวจสอบกิจกรรมล่าสุดของระบบ

## 4. หมวดข้อมูลหลัก

หมวดข้อมูลหลักประกอบด้วย 4 หน้าย่อยที่เป็นฐานข้อมูลพื้นฐานของระบบ ข้อมูลเหล่านี้ต้องถูกต้องครบถ้วนก่อนจึงจะสามารถใช้งานฟีเจอร์อื่น ๆ ของระบบได้อย่างมีประสิทธิภาพ

### 4.1 การจัดการสาขาวิชา

สาขาวิชาเป็นโครงสร้างระดับสูงสุดในการจัดกลุ่มนักศึกษา ตัวอย่างเช่น สาขาเทคโนโลยีสารสนเทศ หรือสาขาวิศวกรรมซอฟต์แวร์

การเพิ่มสาขาใหม่ทำได้โดยไปที่เมนูข้อมูลหลักและเลือกสาขาวิชา กดปุ่มเพิ่มสาขาวิชา กรอกรหัสสาขาเป็นตัวย่อ เช่น IT หรือ SE และชื่อสาขาแบบเต็ม แล้วกดบันทึก การแก้ไขทำได้โดยกดปุ่มแก้ไขในแถวที่ต้องการ แก้ไขข้อมูล แล้วกดบันทึก ระบบไม่อนุญาตให้ลบสาขาวิชาที่ยังมีกลุ่มเรียนหรือนักศึกษาอ้างอิงอยู่

### 4.2 การจัดการกลุ่มเรียน

กลุ่มเรียนเป็นโครงสร้างย่อยภายใต้สาขาวิชา ใช้จัดกลุ่มนักศึกษาตามชั้นปีและกลุ่มการเรียน เพื่อให้อาจารย์สามารถเพิ่มนักศึกษาทั้งกลุ่มเข้าตารางสอนได้พร้อมกัน

การเพิ่มกลุ่มเรียนทำได้โดยไปที่เมนูข้อมูลหลักและเลือกกลุ่มเรียน กดปุ่มเพิ่มกลุ่มเรียน กรอกชื่อกลุ่มเช่น IT66-1 หรือ SE65-2 และเลือกสาขาวิชาที่กลุ่มเรียนนี้สังกัด แล้วกดบันทึก

### 4.3 การจัดการรายวิชา

รายวิชาเป็นข้อมูลสำคัญที่เชื่อมโยงอาจารย์กับตารางสอน การกำหนดรายวิชาในหน้านี้จะทำให้อาจารย์คนที่ถูกกำหนดสามารถสร้างตารางสอนและจัดการนักศึกษาในวิชานั้นได้ทันทีหลังบันทึก

การเพิ่มรายวิชาทำได้โดยไปที่เมนูข้อมูลหลักและเลือกรายวิชา กดปุ่มเพิ่มรายวิชา กรอกรหัสวิชาเช่น 04-000-201 ชื่อวิชา จำนวนหน่วยกิต และเลือกอาจารย์ผู้สอนจากรายชื่ออาจารย์ในระบบ สามารถกำหนดอาจารย์ได้มากกว่า 1 คน แล้วกดบันทึก

### 4.4 การจัดการเหตุผลการเบิก

เหตุผลการเบิกกุญแจคือตัวเลือกที่แสดงบนหน้าจอเครื่อง Kiosk เมื่อนักศึกษาหรืออาจารย์ทำรายการเบิก นักศึกษาต้องเลือกเหตุผลก่อนยืนยัน ซึ่งจะถูกบันทึกลงในประวัติการทำรายการ

การเพิ่มเหตุผลใหม่ทำได้โดยไปที่เมนูข้อมูลหลักและเลือกเหตุผลการเบิก กดปุ่มเพิ่มเหตุผล กรอกชื่อเหตุผล เช่น เรียนปฏิบัติ หรือกิจกรรมชมรม ตั้งสถานะเป็นใช้งานเพื่อให้แสดงบนเครื่อง Kiosk แล้วกดบันทึก

หากต้องการซ่อนเหตุผลชั่วคราวโดยไม่ต้องลบ ให้เปลี่ยนสถานะเป็นปิดใช้งาน เหตุผลนั้นจะไม่แสดงบนเครื่อง Kiosk จนกว่าจะเปิดใช้งานอีกครั้ง

## 5. หน้าจัดการกุญแจ

หน้านี้เป็นหน้าสำคัญที่สุดในการกำหนดและบริหารข้อมูลกุญแจแต่ละดอกในตู้ ข้อมูลที่ถูกต้องในหน้านี้มีผลโดยตรงต่อการทำงานของฮาร์ดแวร์

![ภาพที่ ก.3-2 หน้าจัดการกุญแจ แสดงรายการกุญแจทั้งหมดในตู้](/Users/surachartlimrattanaphun/.gemini/antigravity/brain/ae421eca-c2bb-4a49-9e4a-94a01b6e3aed/kms_keys_page_1776652969481.png)

### 5.1 ความสัมพันธ์ระหว่างข้อมูลและฮาร์ดแวร์

แต่ละรายการในหน้าจัดการกุญแจแสดงข้อมูล 5 ประเภทดังนี้

หมายเลขสล็อต คือหมายเลขช่องสล็อตในตู้ฮาร์ดแวร์ที่ตรงกับหมายเลขวงจรรีเลย์บน Raspberry Pi เมื่อระบบสั่งปลดล็อคสล็อต X ระบบจะส่งสัญญาณไปที่รีเลย์หมายเลข X โดยตรง

รหัสห้อง คือรหัสห้องที่กุญแจดอกนั้นใช้เปิด รหัสต้องตรงกับรหัสที่ใช้ในตารางเรียน

NFC UID คือรหัสเฉพาะของชิป NFC ที่ติดไว้ในพวงกุญแจแต่ละดอก ใช้สำหรับการตรวจสอบการคืนกุญแจ

ชื่อรายการ คือคำอธิบายทรัพย์สิน เช่น กุญแจห้องปฏิบัติการเครือข่าย 44-703

สถานะ แสดงว่ากุญแจอยู่ในตู้หรือถูกยืม

### 5.2 การเพิ่มกุญแจ

ไปที่เมนูจัดการกุญแจ กดปุ่มเพิ่มกุญแจ กรอกข้อมูลดังนี้

ระบุหมายเลขสล็อตที่ตรงกับวงจรรีเลย์บนบอร์ดฮาร์ดแวร์ ตรวจสอบจากแผนผังวงจรของตู้ก่อนกรอก หากกรอกผิดระบบจะปลดล็อคสล็อตผิดดอก ระบุรหัสห้องที่กุญแจดอกนี้ใช้เปิด รหัสต้องตรงกับรหัสที่ใช้ในตารางเรียน นำพวงกุญแจที่มีชิป NFC มาสแกนกับเครื่องอ่าน NFC แบบ USB แล้วนำรหัสที่ได้มากรอกในช่อง NFC UID และระบุชื่อรายการที่ชัดเจน กดบันทึก ระบบอัปเดตข้อมูลในฐานข้อมูลและส่งคำสั่งปรับปรุงข้อมูลไปยัง Raspberry Pi ผ่าน Socket.io ทันที

### 5.3 การแก้ไขกุญแจ

กรณีที่ต้องเปลี่ยนพวงกุญแจพร้อมชิป NFC ใหม่เนื่องจากกุญแจหายหรือชำรุด ให้กดปุ่มแก้ไขในแถวที่ต้องการ อัปเดตค่า NFC UID เป็นรหัสของชิปใหม่ แล้วกดบันทึก ระบบอัปเดตทันที

## 6. หน้าอนุญาตเบิกกุญแจ

หน้าอนุญาตเบิกกุญแจเป็นหน้าที่เจ้าหน้าที่ต้องใช้งานทุกวันทำการ เป็นขั้นตอนสุดท้ายก่อนที่นักศึกษาจะสามารถเบิกกุญแจหน้าตู้ได้

![ภาพที่ ก.3-3 หน้าอนุญาตเบิกกุญแจ แสดงสิทธิ์รายวัน](/Users/surachartlimrattanaphun/.gemini/antigravity/brain/ae421eca-c2bb-4a49-9e4a-94a01b6e3aed/kms_authorize_page_1776653102184.png)

### 6.1 ตัวกรองข้อมูล

หน้านี้มีตัวกรองสองประเภท ตัวกรองวันที่ ให้เลือกวันที่ต้องการดูสิทธิ์ โดยค่าเริ่มต้นคือวันปัจจุบัน และตัวกรองห้อง ให้กรองแสดงเฉพาะสิทธิ์ของห้องที่ต้องการ

### 6.2 การซิงค์สิทธิ์จากตารางสอน

ไปที่หน้าอนุญาตเบิกกุญแจ ตรวจสอบให้แน่ใจว่าตัวกรองวันที่แสดงวันปัจจุบัน เลือกแท็บจากตารางสอน กดปุ่มซิงค์วันนี้ ระบบดำเนินการอ่านตารางเรียนทั้งหมดที่ตรงกับวันในสัปดาห์ของวันนั้นโดยอัตโนมัติ สร้างรายการสิทธิ์สำหรับนักศึกษาทุกคนในแต่ละตาราง และบันทึกข้อมูลรายการสิทธิ์ลงในฐานข้อมูล รายการสิทธิ์ปรากฏในตารางพร้อมข้อมูลห้อง รหัสนักศึกษา ชื่อ วิชา และช่วงเวลา

เจ้าหน้าที่ควรดำเนินการซิงค์ทุกเช้าของวันทำการก่อนเวลา 08:00 น. เพื่อให้นักศึกษามีสิทธิ์เบิกกุญแจได้ตั้งแต่เริ่มคาบเรียน

### 6.3 การเพิ่มสิทธิ์แบบกำหนดเอง

ใช้สำหรับกรณีพิเศษที่ไม่มีในตารางเรียน เช่น กิจกรรมพิเศษหรือการฝึกอบรม เลือกแท็บเพิ่มเอง กดปุ่มเพิ่มสิทธิ์ ค้นหาและเลือกผู้ได้รับสิทธิ์ ระบุรหัสห้อง เลือกวันที่ และกำหนดช่วงเวลาที่อนุญาตให้เบิกในหน่วย 15 นาที กดเพิ่มสิทธิ์

การลบสิทธิ์ทำได้โดยกดปุ่มลบสีแดงในแถวที่ต้องการ แล้วยืนยันการลบ สิทธิ์จะถูกยกเลิกทันที

## 7. หน้าจัดการผู้ใช้งาน

เจ้าหน้าที่สามารถจัดการบัญชีผู้ใช้ทุกประเภทในระบบ การจัดการแบ่งออกเป็นสามหน้าย่อย ได้แก่ นักศึกษา อาจารย์ และเจ้าหน้าที่

![ภาพที่ ก.3-4 หน้าจัดการผู้ใช้งานนักศึกษา](/Users/surachartlimrattanaphun/.gemini/antigravity/brain/ae421eca-c2bb-4a49-9e4a-94a01b6e3aed/kms_students_page_v2_1776652954683.png)

### 7.1 การค้นหาและกรองผู้ใช้

ในแต่ละหน้าย่อยมีช่องค้นหาที่ให้พิมพ์ชื่อ นามสกุล อีเมล หรือรหัสประจำตัว ระบบกรองผลลัพธ์ทันทีโดยไม่ต้องกดปุ่ม

### 7.2 การเพิ่มนักศึกษา

ไปที่เมนูจัดการผู้ใช้งานและเลือกนักศึกษา กดปุ่มเพิ่มนักศึกษา กรอกชื่อ นามสกุล รหัสนักศึกษา อีเมล สาขาวิชาและกลุ่มเรียนที่สังกัด และรหัสผ่านเริ่มต้น แล้วกดบันทึก

หลังเพิ่มข้อมูลในระบบแล้ว นักศึกษาต้องมาลงทะเบียนภาพใบหน้าที่เครื่อง ZKTeco SmartAC1 ในอาคาร เพื่อให้สามารถสแกนใบหน้าที่เครื่อง Kiosk ได้

### 7.3 การเพิ่มอาจารย์

ไปที่เมนูจัดการผู้ใช้งานและเลือกอาจารย์ กดปุ่มเพิ่มอาจารย์ กรอกชื่อ นามสกุล รหัสพนักงาน อีเมล และรหัสผ่านเริ่มต้น แล้วกดบันทึก

หลังเพิ่มอาจารย์แล้ว เจ้าหน้าที่ต้องกำหนดรายวิชาให้อาจารย์ในหน้าจัดการรายวิชาก่อน อาจารย์จึงจะสามารถสร้างตารางสอนได้

### 7.4 การแก้ไขและลบผู้ใช้

การแก้ไขทำได้โดยกดปุ่มแก้ไขในแถวผู้ใช้ที่ต้องการ แก้ไขข้อมูลในแบบฟอร์ม แล้วกดบันทึก ข้อมูลที่แก้ไขได้ ได้แก่ ชื่อ นามสกุล อีเมล สาขาวิชา กลุ่มเรียน และรหัสผ่าน

การลบผู้ใช้ทำได้โดยกดปุ่มลบในแถวที่ต้องการ กล่องยืนยันแสดงขึ้น กดยืนยันเพื่อดำเนินการ หากต้องการเพียงหยุดไม่ให้ผู้ใช้เข้าสู่ระบบ ควรเปลี่ยนรหัสผ่านของบัญชีนั้นให้ไม่สามารถใช้ได้แทนการลบ

## 8. หน้าประวัติการเบิก-คืน

หน้านี้แสดงประวัติการทำรายการทั้งหมดของระบบ ใช้สำหรับการตรวจสอบ การหาข้อมูลเพื่อตัดสินใจ และการแก้ไขข้อมูลที่ไม่ถูกต้อง

![ภาพที่ ก.3-5 หน้าประวัติการเบิก-คืน](/Users/surachartlimrattanaphun/.gemini/antigravity/brain/ae421eca-c2bb-4a49-9e4a-94a01b6e3aed/kms_bookings_page_1776653008047.png)

### 8.1 ข้อมูลในรายการประวัติ

แต่ละรายการแสดงข้อมูลดังต่อไปนี้ ชื่อ นามสกุล และรหัสประจำตัวของผู้ใช้ รหัสห้องที่ยืม สถานะซึ่งได้แก่ กำลังยืม คืนแล้ว หรือเกินกำหนด วันที่และเวลาที่ทำรายการเบิก วันที่และเวลาที่คืน และเหตุผลที่ระบุเมื่อทำรายการเบิก

### 8.2 การกรองและค้นหา

รองรับการกรองข้อมูลด้วยช่วงวันที่ สถานะ รหัสห้อง และชื่อหรือรหัสผู้ใช้ได้พร้อมกัน

### 8.3 การแก้ไขสถานะรายการด้วยตนเอง

กรณีที่นักศึกษาคืนกุญแจแล้วแต่ระบบไม่อัปเดตสถานะ เช่น ชิป NFC บนพวงกุญแจชำรุด เจ้าหน้าที่ค้นหารายการที่ต้องการแก้ไข กดปุ่มแก้ไขในแถวนั้น เปลี่ยนสถานะเป็นคืนแล้ว ระบุเวลาคืนที่ถูกต้อง กดบันทึก การแก้ไขทุกครั้งจะถูกบันทึกลงใน Audit Log พร้อมบัญชีเจ้าหน้าที่ที่ดำเนินการ

## 9. หน้าจัดการ Penalty

หน้านี้แสดงรายการบทลงโทษที่ระบบสร้างอัตโนมัติเมื่อนักศึกษาคืนกุญแจเกินกำหนดเวลา

![ภาพที่ ก.3-6 หน้าจัดการ Penalty](/Users/surachartlimrattanaphun/.gemini/antigravity/brain/ae421eca-c2bb-4a49-9e4a-94a01b6e3aed/kms_penalty_page_v2_1776653072315.png)

### 9.1 การดูรายการ Penalty

แต่ละรายการแสดงชื่อและรหัสนักศึกษา ห้องที่ยืม เวลากำหนดคืน เวลาที่คืนจริง ระยะเวลาที่เกิน และจำนวนคะแนนที่หัก สามารถกรองข้อมูลตามช่วงวันที่หรือค้นหาตามชื่อและรหัสนักศึกษาได้

### 9.2 การยกเว้น Penalty

กรณีที่นักศึกษาแจ้งเหตุสุดวิสัย เช่น ตู้กุญแจเกิดขัดข้องหรือระบบออฟไลน์ชั่วคราว เจ้าหน้าที่สามารถพิจารณายกเว้นบทลงโทษตามดุลยพินิจ โดยค้นหารายการ Penalty ในตาราง กดปุ่มลบหรือยกเว้น ระบุเหตุผลในช่องบันทึก ซึ่งข้อมูลนี้จะถูกบันทึกลงใน Audit Log โดยอัตโนมัติ กดยืนยัน คะแนนหรือสิทธิ์ของนักศึกษาจะได้รับการคืนให้ทันที

เจ้าหน้าที่ใช้งานเครื่อง Kiosk ได้เช่นเดียวกับอาจารย์ โดยสแกนใบหน้าผ่านกล้อง ZKTeco เพื่อยืนยันตัวตน รวมถึงมีสิทธิ์เบิกกุญแจนอกตารางเรียน รายละเอียดการใช้งานฟังก์ชันทั้ง 5 ให้อ้างอิงตามภาคผนวก ก-1 และภาคผนวก ก-2

## 11. แนวปฏิบัติรายวันสำหรับเจ้าหน้าที่

### 11.1 ก่อนเริ่มเวลาทำการ

เปิดเครื่องคอมพิวเตอร์และเข้าสู่ระบบ Web Dashboard ตรวจสอบหน้า Dashboard เพื่อดูสถานะกุญแจโดยรวม หากพบกุญแจที่มีสถานะถูกยืมค้างมาตั้งแต่วันก่อน ให้ตรวจสอบและดำเนินการแก้ไข จากนั้นดำเนินการซิงค์ตารางเรียนในหน้าอนุญาตเบิกกุญแจเพื่อสร้างสิทธิ์ประจำวัน

### 11.2 ระหว่างวันทำการ

ตอบสนองต่อการร้องขอของนักศึกษาหรืออาจารย์ที่ไม่สามารถเบิกกุญแจได้ โดยตรวจสอบสิทธิ์และดำเนินการแก้ไขตามความเหมาะสม ติดตามสถานะกุญแจที่เกินกำหนดและประสานงานกับผู้เกี่ยวข้อง

### 11.3 สิ้นวันทำการ

ตรวจสอบรายการที่มีสถานะกำลังยืมและเกินกำหนดทั้งหมด ดำเนินการแก้ไขรายการที่มีปัญหา บันทึกเหตุการณ์ผิดปกติ และออกจากระบบอย่างปลอดภัย

## 12. การบำรุงรักษาระบบและฮาร์ดแวร์

### 12.1 การตรวจสอบสถานะระบบ

เจ้าหน้าที่ควรตรวจสอบสถานะระบบเป็นประจำสัปดาห์ละครั้ง ดังนี้

ตรวจสอบการเชื่อมต่อระหว่าง Backend Server และ Raspberry Pi โดยดูจากหน้า Dashboard หากการ์ดแสดงข้อมูลแบบเรียลไทม์ถูกต้อง แสดงว่าการเชื่อมต่อปกติ

ตรวจสอบสถานะกุญแจทั้งหมดในหน้าจัดการกุญแจว่าไม่มีกุญแจที่แสดงสถานะถูกยืมค้างนานผิดปกติ

ทดสอบชิป NFC บนพวงกุญแจโดยนำพวงกุญแจเสียบเข้าช่องสล็อตแล้วตรวจสอบว่าระบบตรวจจับได้

ทดสอบกล้องสแกนใบหน้า ZKTeco โดยสแกนใบหน้าของเจ้าหน้าที่เองที่เครื่อง Kiosk

### 12.2 การแก้ปัญหาที่พบบ่อย

ปัญหา กุญแจไม่ปลดล็อคหลังนักศึกษายืนยันการเบิก การวินิจฉัย ให้ตรวจสอบตามลำดับดังนี้ ตรวจสอบว่า Backend Server ทำงานปกติ ตรวจสอบว่า Raspberry Pi เชื่อมต่ออยู่กับ Backend ผ่าน Socket.io ตรวจสอบว่าหมายเลขสล็อตในหน้าจัดการกุญแจตรงกับวงจรฮาร์ดแวร์จริง และตรวจสอบว่าสายไฟของรีเลย์ที่สล็อตนั้นยังแน่นดี

ปัญหา นักศึกษาสแกนใบหน้าแล้วระบบไม่จดจำทั้งที่เคยใช้งานได้ การวินิจฉัย ตรวจสอบว่าเครื่อง ZKTeco เชื่อมต่อกับเครือข่ายและ Backend อยู่ อาจจำเป็นต้องถ่ายภาพใบหน้าใหม่หากคุณภาพภาพเดิมไม่ดีพอ

ปัญหา ระบบออฟไลน์ หน้าจอ Kiosk แสดงข้อความไม่สามารถเชื่อมต่อ การวินิจฉัย ตรวจสอบการเชื่อมต่ออินเทอร์เน็ตของเครื่อง Kiosk และตรวจสอบว่า Backend Server ยังทำงานปกติ

### 12.3 ความปลอดภัยของระบบ

เจ้าหน้าที่ควรลบบัญชีของนักศึกษาที่จบการศึกษาหรือลาออกออกจากระบบ และลบบัญชีของอาจารย์หรือเจ้าหน้าที่ที่ย้ายหรือลาออกทันทีภายหลังที่ทราบข้อมูล เพื่อป้องกันการใช้สิทธิ์ที่ไม่ชอบด้วยกฎหมาย
